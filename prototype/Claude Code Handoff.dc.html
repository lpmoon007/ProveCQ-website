⚠ First action: retire FPO as the front door — migrate, don't discard

The FPO brand and the FPO-branded homepage/services/pricing are deprecated as positioning — Prove is the front door now. But do not throw the FPO work away. Two moves:

  • Migrate the FPO landing/keyword pages into provecq.com as SEO funnel content (a /resources or /guides layer): Landing-*.dc.html, Lifecycle.dc.html, Resources.dc.html. They capture real "fractional CHRO / people officer / HR" search demand that "Commitment Quotient" will never rank for. Re-point every CTA from "book an FPO audit / download the Playbook" to "see how Prove works / book a call," and restyle into the Prove design system. FPO-as-a-service survives only as the premium done-for-you delivery tier of Prove.
  • 301 redirect FPO.com into provecq.com so the domain's demand flows into Prove.
  • Retire the FPO-branded shell pages (Home, Services, GetToGreen*, Playbook, Team, Contact, old Claude Code Handoff) — replaced by the provecq/ versions. Keep the design craft, the Heatmap pattern, and gtg-data.js.

1. Positioning (do not drift from this)

  • Lead with the outcome, never the tool. Hero = "Stop betting on the wrong people." CQ/Prove is the mechanism, introduced second. Never open a page with "Commitment Quotient is an assessment."
  • Buyer: founder / CEO of a scaling SMB (esp. EOS-run). Always sold to the founder.
  • Wedge: "Prove before you promote" + the EOS angle ("make the Capacity box real"). EOS Implementers are a channel.
  • It's behavioral measurement, not psychometric. Keep outputs framed as decision-support for the leader, never auto-hire/fire. (Removes legal exposure.)
  • Naming: the trait is "Applied Grit" — never "Grit" alone (too closely tied to Duckworth). Abbreviate as "A. Grit" only where space forces it.
  • Brand spine: Be Legendary is the house; Prove/CQ is the product; "Prove" is the spoken brand; Commitment Quotient™ is the named methodology inside.

4. The product: CQ Heatmap + Prove

The hero asset is the CQ Heatmap (see provecq/Heatmap.dc.html for the interaction): a team roster scored on three behaviors, with a per-person CQ score, tier, recommended founder action, and risk note. Build it as the authed product. Authoritative scoring source: follow uploads/CQ_Scoring_System_Technical_Specification_Current_State.docx — it governs over the demo numbers below.

// PRODUCTION model (per the Technical Spec) — trait + composite on 0-100
Person = { name, title,
  initiative_score: 0-100,      // store independently
  applied_grit_score: 0-100,    // "Applied Grit" in data model; "Grit" = display only
  learnability_score: 0-100,
  cq_composite: 0-100,          // versioned weighted_mean, full precision, round on display
  scoring_profile_id, version,  // e.g. "cq-core-2026-demo" v0.1.0 (status: provisional)
  tier, founderAction, riskNote }

// Composite (provisional profile "cq-core-2026-demo", NOT permanent doctrine):
CQ = Initiative*0.40 + Applied_Grit*0.35 + Learnability*0.25   // weights sum to 1.0
// Weights/thresholds live in a versioned scoring PROFILE, never hard-coded.
// Published profiles are immutable; store raw responses + version for reproducibility.
// The prototype Heatmap/ROI use a simplified 0-10 display scale — cosmetic only.

  • Migration blockers (from the spec): the assessment item bank, item-to-trait mapping, reverse-scoring rules, norms/percentiles, and the 7 archetype thresholds were not recovered. Do not invent them — keep archetypes/percentile claims disabled until source rules are supplied. Do not represent CQ as norm-referenced or clinically validated beyond available evidence.
  • Prove can run self-directed (like Be Legendary's Mindset Gym) — build the async challenge/tracking engine so delivery isn't gated by a facilitator. This is the scalability unlock.
  • Multi-tenant + auth (Clerk/Auth.js), Postgres, per-company Heatmaps, cycle history. The behavioral dataset that accumulates is the long-term moat — design the schema to capture and benchmark it.
  • Better Every Day = the recurring subscription for high-CQ players (weekly tracking, monthly commitments, drift alerts); gated by passing Prove. Per-seat billing (Stripe).
  • EOS mode: map CQ output to the People Analyzer / GWC "Capacity" and Quarterly Conversations.

6. The free-tool funnel (build the ladder)

Three gated/ungated free tools feed the pipeline. Wire them as one ladder that always ends at "book a call":

  • Certainty Diagnostic (Diagnostic.dc.html) — ungated 10-question quiz → instant Certainty Score. Result CTA → Get to Green + book a call. (Optionally capture email to "send my results.")
  • Hidden Gem Identifier (HiddenGem.dc.html) — HubSpot-gated (form id 7f180c70-7e33-4570-8d3e-b37de009dfd3, portal 20276071). Mount the HubSpot embed OUTSIDE the React tree (see the DC's imperative mount) or via a real form component in the rebuild.
  • Get to Green (GetToGreen.dc.html) — fully open interactive sample (real 146-item schema from gtg-data.js, category-weighted). The personalized map is behind a real self-serve account (auth + database): user signs up automatically, scores/saves their own company's map, and manages it themselves — no manual provisioning by the client. This free account is the wedge that upgrades to Prove/BED. Persist per-company scores, status history, and owners; this accumulating data is also part of the moat.
  • Free Tools hub (FreeTools.dc.html) — the container for all three; linked in nav. Homepage has a "Start free" band. Every tool → book a call.
  • Lead magnet copy + the "It's the Metric" manifesto (Metric.dc.html) and 15 guide pages are the GEO/SEO top-of-funnel that route into these tools.
Suggested build order: (1) marketing site + Guides/SEO layer → (2) the free tools (Diagnostic, 4 AM Test, Hidden Gem, Get to Green sample) → (3) the self-serve Get to Green account (auth + DB) → (4) the Prove engine (Snapshot / Validation / Decision + Heatmap) → (5) Better Every Day subscription + billing. Ship 1–2 to start capturing leads while 3–5 are built.
